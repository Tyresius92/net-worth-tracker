# Text

Text handles all non-heading typography — body copy, standfirsts, metadata, labels, and inline code. Every variant is set in Source Serif 4 except `code`, which uses the system monospace stack.

## When to use

Use Text for content the reader is meant to read or scan: paragraphs, captions, date straps, authorship lines. If the content names a section or page, use [`Heading`](?path=/docs/components-heading--docs) instead.

## Deck

The standfirst. One sentence that bridges the Level 1 heading and the body copy below it. Use it once per page, directly below the H1. Two sentences maximum.

<Canvas of={TextStories.Deck} />

## Body

The workhorse. Use for paragraphs the reader actually reads — editorial copy, full-sentence error messages, long-form content.

<Canvas of={TextStories.Body} />

## Byline

Free-floating metadata on its own line — author attribution, date straps, footer copyright. Rendered in uppercase with widened tracking.

<Canvas of={TextStories.Byline} />

## Caption

A small label paired with a value directly beside or below it. Rendered in uppercase with tighter tracking than byline, because it is always part of a key/value pair. Defaults to a `<span>` so it sits inline with its value.

<Canvas of={TextStories.Caption} />

## Code

Inline code, token names, shell commands. Uses the system monospace stack. Appears rarely in the application itself — its primary purpose is in design system documentation.

<Canvas of={TextStories.Code} />

## Drop cap

Pass `dropCap` to float the first letter as a large typographic initial — a newspaper convention for the opening paragraph of a long-form piece.

<Canvas of={TextStories.DropCap} />

Use only on the first paragraph of a section and only with `body` copy. Do not combine with `byline`, `caption`, or `deck` — the first-letter sizing assumes normal body line-height.

## Overriding the element

Each variant has a sensible default HTML element: `deck`, `body`, and `byline` render `<p>`; `caption` renders `<span>`; `code` renders `<code>`. Use the `as` prop when the default element does not fit the surrounding structure.

<Canvas of={TextStories.AsSpan} />

A common case is rendering a `body` as a `<span>` when it appears inside a flex container that cannot contain a block element, or a `caption` as a `<p>` when it stands alone rather than beside a value.
