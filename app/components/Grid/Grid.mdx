# Grid

Grid is the structural layout primitive for multi-column page composition. It handles the relationship between regions — where something sits on the page and how much horizontal space it occupies. Spacing and visual treatment within each region are handled by [`Box`](?path=/docs/components-box--docs).

## When to use

- Arranging distinct content regions side by side (main content + sidebar, editorial columns)
- Layouts that need to respond to viewport width — different column counts at different sizes
- Any situation where items need to align to a shared column structure across rows

## When not to use

**Spacing within a component:** use `Box` with padding props. Grid is for structure between regions, not for internal component layout.

**Simple side-by-side elements:** if two items just need to sit next to each other and don't require column alignment, `Box` with `display="flex"` is less overhead.

**Named grid areas or explicit row placement:** Grid intentionally does not support these. Reach for a CSS class if a layout genuinely requires them.

## The 12-column model

Grid divides its container into 12 equal columns. 12 was chosen because it divides evenly by 2, 3, 4, and 6 — covering the full range of common editorial fractions without the cognitive overhead of a finer scale.

## Offset

The `offset` props shift a `Grid.Item`'s starting column without inserting an empty DOM element. An offset of `3` means "start at column 4" — the item begins three columns in from the left edge.

Offset props follow the same breakpoint naming convention as span props: `xsOffset`, `sOffset`, `mOffset`, `lOffset`, `xlOffset`. They also carry upward — `mOffset={3}` applies at medium, large, and extra-large unless overridden.

Setting an offset to `0` explicitly resets it back to auto-placement, which cancels a carry-upward offset at a specific breakpoint.

## Wrapping

When items in a row exceed 12 columns combined, the overflow wraps to a new row automatically. This is standard CSS Grid behavior — no extra configuration needed.

## Ordering

The `order` props (`xsOrder`, `sOrder`, `mOrder`, `lOrder`, `xlOrder`) change the visual position of items within the grid without altering the DOM order. This is useful when the most important content should lead visually on mobile but occupy a secondary column position on wider screens.

Order values carry upward. Setting `xsOrder={1}` on an item makes it appear first on all breakpoints unless a larger-breakpoint order prop overrides it.

**Accessibility note:** `order` is visual only. It does not change tab order or the sequence in which screen readers announce content. If a different reading order is meaningful for the content, the DOM order itself should reflect it.

## Gaps

`Grid` accepts `gap` (column gap) and `rowGap` (row gap) props, both using the design token spacing scale. Column gap and row gap are independent — it is common to want tighter row gaps than column gaps, or no row gap at all.

## Grid and Box together

Grid handles placement; Box handles everything inside a placed region. A typical pattern wraps each `Grid.Item`'s content in a `Box` to apply padding or a background:

```tsx
<Grid gap={24} rowGap={16}>
  <Grid.Item m={8}>
    <Box p={16} bg="sand-2">
      Main content
    </Box>
  </Grid.Item>
  <Grid.Item m={4}>
    <Box p={16}>
      Sidebar
    </Box>
  </Grid.Item>
</Grid>
```
