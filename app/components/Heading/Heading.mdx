# Heading

Heading establishes hierarchy. It names a page, a section, or a subsection — always in Libre Baskerville, always in `--color-sand-12`. The three levels correspond directly to `<h1>`, `<h2>`, and `<h3>` in the DOM.

## When to use

Use Heading whenever you need to label a block of content that the reader will navigate to or scan past. Each level has a specific role:

- **Level 1** — the primary fact on the page. One per page. On the dashboard: the net worth figure. On a settings screen: the screen's name.
- **Level 2** — a section headline. Names a block of related content within the page (Highlights, Open accounts, Net worth history).
- **Level 3** — a subsection or aside title. Used inside cards or within longer sections. The Specifications sidebar is a Level 3.

## When not to use

Do not use Heading to make text larger or bolder for visual effect. If you need a label paired with a value (STACK, AUTH), use [`Text role="caption"`](?path=/docs/components-text--docs) instead. If you need free-floating metadata (a date strap, an author line), use `Text role="byline"`.

**Never skip heading levels.** A Level 3 must follow a Level 2; a Level 2 must follow a Level 1. If you find yourself wanting a fourth level, consider restructuring the content first — a fourth level is rarely necessary and often signals that a section should be split or that a non-heading role (`caption`, `byline`) would serve better. If the hierarchy genuinely requires a fourth level, it can be designed and added to the `Heading` component as `level={4}`.

## Font size override

In rare cases — large editorial headlines, hero text — the level's default size is too small for the visual context. Pass `fontSize` to override it with any value from the token scale.

```tsx
<Heading level={1} fontSize={88}>
  A Ledger of Your Own.
</Heading>
```

<Canvas of={HeadingStories.FontSizeOverride} />

Use this sparingly. `fontSize` changes the visual presentation without changing the semantic level — a Level 1 with `fontSize={88}` is still an `<h1>` and still counts as your one-per-page primary heading. Do not use it to make a lower-level heading look like a higher one; use the correct level and adjust the design instead.

## Accessibility

`Heading` renders the correct semantic element for each level — `<h1>` through `<h3>` — so the document outline is always accurate. Screen readers use heading levels to let users skip between sections; a correct heading hierarchy is what makes that navigation possible.

Avoid using `level` to achieve a visual size you want on an element that is not actually a heading. If a heading-like size is needed for non-heading content, revisit the role against the typography system.
