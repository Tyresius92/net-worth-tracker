# Box

Box is the universal layout primitive for Broadsheet. All spacing (margin and padding), background color, border color, and flex layout in the application should be handled through Box props — not bespoke CSS classes or inline style objects.

## When to use

- Adding margin or padding to any element
- Applying a background color or border from the token system
- Creating a flex container

## When not to use

Box is a block layout primitive. Do not use it for inline content — `<span>`, `<a>`, `<strong>`, and similar inline elements should not be added to the `is` union and Box should not be used as a substitute for them.

If you need a block element that `is` doesn't yet support, add it to the union rather than nesting a Box inside a plain HTML element.

## Polymorphism

Box defaults to rendering as a `<div>`, but can render as `<article>` or `<section>` via the `is` prop. Use this when the content has a meaningful document role that one of those elements expresses.

```tsx
<Box is="article" p={16}>
  <h2>Chase Checking</h2>
  ...
</Box>
```

Adding further **block-level** elements to the `is` union is acceptable as the need arises. **Inline elements** (e.g. `<span>`, `<a>`) should not be added — Box is a block layout primitive and its spacing model assumes block formatting context.

## Spacing

Box accepts margin and padding props that map to the design token scale. Props cascade from least to most specific — a more specific prop overrides a less specific one.

```
p  ──▶  px, py
        │    │
        ▼    ▼
       pl,  pt,
       pr   pb
```

Setting `p={16}` applies `16px` to all four sides. Setting `p={16} px={8}` overrides both horizontal sides to `8px`. Setting `p={16} px={8} pl={4}` additionally overrides only the inline-start side to `4px`.

## Color

Box accepts `bg` and `color` props, each taking a token name from the design system's color palette. In practice these are `sand-N` values — see the [Color](?path=/docs/color--docs) doc for guidance on which steps to use when.

## Borders

Box accepts `border`, `borderX`, `borderY`, `borderTop`, `borderRight`, `borderBottom`, and `borderLeft` props, each taking a `StrokeValue` object with a required `color` and optional `width` (1–3) and `style` (`"solid"`, `"dotted"`, `"dashed"`).

```tsx
// All four sides
<Box border={{ color: "sand-7" }} p={12}>…</Box>

// Horizontal sides only
<Box borderX={{ color: "sand-6" }} p={12}>…</Box>

// One side, custom width
<Box borderLeft={{ color: "sand-8", width: 3 }} pl={12}>…</Box>
```

Props cascade from least to most specific — the same pattern as padding. `border` sets all four sides; `borderX`/`borderY` override horizontally or vertically; individual side props override everything.

```
border  ──▶  borderX, borderY
              │           │
              ▼           ▼
         borderLeft,  borderTop,
         borderRight  borderBottom
```

## Flex layout

Setting `display="flex"` on Box enables all flexbox container and item props. This is the correct way to create flex layouts — do not reach for a CSS class.

```tsx
<Box display="flex" gap={8} alignItems="center" justifyContent="space-between">
  <span>Left</span>
  <span>Right</span>
</Box>
```

Flex item props (`flexGrow`, `flexShrink`, `flexBasis`, `alignSelf`) can be applied to a child Box to control how it behaves within a flex container.

```tsx
<Box display="flex" gap={8}>
  <Box flexGrow={1}>Fills remaining space</Box>
  <Box flexShrink={0}>Fixed size</Box>
</Box>
```

## Multi-column layout

Setting `columnCount` on Box enables CSS multi-column layout. Use `columnGap` to control the gutter width and `columnRule` to draw a dividing line between columns. This is appropriate for flowing editorial text — do not use it for grid or card layouts (use flex for those).

```tsx
<Box columnCount={2} columnGap={24} columnRule={{ color: "sand-6" }}>
  <p>Long-form content that flows across columns…</p>
</Box>
```

<Canvas of={BoxStories.WithColumnLayout} />

`columnCount`, `columnGap`, and `columnRule` are only available together — they are not available on flex or block Box variants. Setting `columnCount` implicitly opts out of `display="flex"`.

## Typography props

`textAlign` and `hyphens` are available on all Box variants and emit directly as CSS properties. Use these when the layout context requires alignment or hyphenation control that would otherwise need a wrapper element.

```tsx
<Box textAlign="justify" hyphens="auto">
  …
</Box>
```

Prefer setting these on the innermost element that needs them rather than on a layout wrapper.

## Accessibility

By default Box renders as a `<div>`, which carries no implicit ARIA role. Use the `is` prop to render a semantically appropriate element when the content warrants it. If the element you need isn't in the `is` union yet, add it.

Box accepts an `id` prop, which can be used to associate it with ARIA attributes on other elements (e.g. `aria-labelledby`, `aria-describedby`).
